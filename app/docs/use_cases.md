# Описание вариантов использования — «Земельные участки»

> Сервис поиска, анализа и управления объявлениями о продаже земельных участков.  
> Роли: **Пользователь**, **Администратор** (наследует все возможности Пользователя).

---

## UC1: Просмотр каталога участков

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Сервис доступен; в базе есть объявления.

**Основной сценарий:**
1. Пользователь открывает страницу «Каталог».
2. Система отображает первую страницу карточек участков (сортировка по дате, убывание). Каждая карточка содержит: миниатюру, название, локацию, цену (₽), площадь (сотки), три мини-индикатора оценки (Инфра, Экология, Характеристики).
3. Пользователь нажимает номер страницы для перехода к другой странице.
4. Пользователь нажимает на карточку — система переходит к просмотру деталей (UC5).

**Альтернативный сценарий:**
- *2a.* Объявлений нет → система показывает «Нет объявлений».
- *2b.* Сервер недоступен → система показывает сообщение об ошибке.

**Результат:** Пользователь видит постраничный список участков и может перейти к деталям любого из них.

---

## UC2: Фильтрация участков

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Пользователь находится на странице каталога (UC1).

**Основной сценарий:**
1. Пользователь нажимает кнопку «Фильтры».
2. Система раскрывает панель с полями: минимальная/максимальная цена (числовое поле, ₽), минимальная/максимальная площадь (числовое поле, сотки), минимальная/максимальная цена за сотку (числовое поле, ₽/сот.), минимальный общий балл (ползунок, 0–1), минимальный балл инфраструктуры (ползунок, 0–1), минимальный балл характеристик (ползунок, 0–1), локация (текстовое поле, поиск по подстроке).
3. Пользователь заполняет нужные поля.
4. Пользователь нажимает «Применить».
5. Система отображает только участки, удовлетворяющие заданным критериям. Активные фильтры отображаются как цветные метки.

**Альтернативный сценарий:**
- *4a.* Пользователь нажимает «Сбросить» → все фильтры очищаются, каталог показывает полную выборку.
- *5a.* Ни один участок не удовлетворяет фильтрам → система показывает «Нет объявлений».

**Результат:** Каталог отображает только участки, соответствующие заданным критериям.

## UC3: Сортировка участков

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Пользователь находится на странице каталога.

**Основной сценарий:**
1. Пользователь видит панель сортировки с кнопками: Дата, Цена, Площадь, ₽/сот., Score, Инфра, Хар-ки.
2. Пользователь нажимает одну из кнопок (например, «Цена»).
3. Система загружает участки, отсортированные по выбранному полю (по возрастанию).
4. Повторное нажатие на ту же кнопку переключает порядок (возрастание ↔ убывание).

**Альтернативный сценарий:**
- *3a.* Сервер недоступен → система показывает сообщение об ошибке, предыдущий список сохраняется.

**Результат:** Участки отображаются в выбранном пользователем порядке.

## UC4: Семантический поиск участков

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Сервис доступен; в базе есть участки с эмбеддингами.

**Основной сценарий:**
1. Пользователь вводит произвольный текстовый запрос в поисковую строку на странице каталога (текстовое поле, например: «участок с газом у леса рядом с метро»).
2. Пользователь нажимает Enter.
3. Система перенаправляет на страницу результатов поиска.
4. Система отображает ранжированный список результатов. Каждый результат содержит: номер позиции, миниатюру, название, локацию, цену (₽), сниппет описания, общий балл (шкала), балл Jina-переранжирования, балл векторного поиска, теги характеристик.
5. Пользователь нажимает на результат — система переходит к просмотру деталей (UC5).

**Альтернативный сценарий:**
- *1a.* Пустой запрос → поиск не выполняется.
- *4a.* Результатов нет → система показывает «Ничего не найдено».
- *4b.* Сервер недоступен → система показывает сообщение об ошибке.

**Результат:** Пользователь получает список участков, семантически релевантных его запросу.

## UC5: Просмотр деталей участка

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Участок существует в базе данных.

**Основной сценарий:**
1. Пользователь нажимает на карточку участка (в каталоге, результатах поиска или на карте).
2. Система загружает и отображает полную информацию об участке:
   - Название, локация, адрес, дата создания/изменения.
   - Миниатюрное изображение.
   - Цена (₽), площадь (сотки), цена за сотку (₽/сот.).
   - Полное текстовое описание.
   - Теги характеристик (газ, электричество, ИЖС и т.д.).
   - Четыре индикатора оценок: Общий балл, Инфраструктура, Экология, Характеристики (0–100%).
   - Таблица расстояний до 8 типов инфраструктуры (Метро, Больница, Школа, Детсад, Магазин, ПВЗ, Автобус, Негативный объект) с цветовой индикацией: зелёный (<5 км), жёлтый (5–15 км), красный (>15 км).
   - Координаты (широта, долгота), идентификатор Avito.
3. Пользователь анализирует информацию.

**Альтернативный сценарий:**
- *2a.* Некорректный идентификатор или участок удалён → система показывает «Не найдено».
- *2b.* Сервер недоступен → система показывает сообщение об ошибке.

**Результат:** Пользователь видит полную информацию об участке, включая аналитические оценки и расстояния до инфраструктуры.

## UC6: Просмотр карты участков

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** В базе данных есть участки с координатами.

**Основной сценарий:**
1. Пользователь нажимает «Карта» в навигации.
2. Система прогрессивно загружает участки порциями по 200 штук и отображает их на интерактивной карте Leaflet в виде круглых маркеров с цветовой индикацией по баллу: зелёный (≥70), жёлтый (40–69), красный (<40).
3. Панель статистики показывает: количество загруженных/всего, прогресс загрузки, средний балл.
4. Пользователь нажимает на маркер — система показывает всплывающее окно с названием, локацией, ценой, площадью, баллом, характеристиками и ссылкой «Подробнее →».
5. Пользователь нажимает «Подробнее →» — система переходит к просмотру деталей (UC5).

**Альтернативный сценарий:**
- *2a.* Участков нет → карта отображается пустой, статистика показывает 0.
- *2b.* Ошибка загрузки → часть маркеров не отображается.

**Результат:** Пользователь видит географическое расположение участков с цветовой индикацией качества.

## UC8: Добавление нового участка

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Пользователь авторизован в системе.

**Основной сценарий:**
1. Пользователь нажимает «Новый участок» в навигации.
2. Система отображает форму создания с интерактивной картой Leaflet.
3. Пользователь нажимает на точку на карте для указания координат участка. Система ставит золотой маркер в выбранной точке.
4. Пользователь заполняет поля формы:
   - **Название** (текстовое поле, обязательное).
   - **Описание** (многострочное текстовое поле).
   - **Цена** (числовое поле, ₽).
   - **Площадь** (числовое поле, сотки, шаг 0.1).
   - **Локация** (текстовое поле, район).
   - **Адрес** (текстовое поле).
   - **Гео-описание** (текстовое поле, например «д. Низино, СНТ Сад-2»).
   - **URL объявления** (текстовое поле, ссылка на оригинал).
   - **URL миниатюры** (текстовое поле, ссылка на изображение).
5. Пользователь нажимает «Добавить участок».
6. Система автоматически рассчитывает характеристики, эмбеддинг, расстояния до инфраструктуры и итоговый балл.
7. Система перенаправляет на страницу деталей созданного участка (UC5).

**Альтернативный сценарий:**
- *5a.* Координаты не указаны → система показывает ошибку «Укажите точку на карте».
- *5b.* Название не заполнено → форма не отправляется (валидация браузером).
- *6a.* Ошибка сервера → система показывает сообщение об ошибке.

**Результат:** В базе данных создан новый участок с автоматически рассчитанными оценками и расстояниями. Участок назначен текущему пользователю как владельцу.

## UC9: Редактирование своего участка

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Пользователь авторизован; участок принадлежит этому пользователю (owner_id совпадает); пользователь находится на странице деталей своего участка (UC5).

**Основной сценарий:**
1. Пользователь нажимает кнопку «Редактировать» на странице деталей участка.
2. Система открывает форму редактирования с предзаполненными текущими значениями полей и картой с текущим маркером координат.
3. Пользователь изменяет нужные поля (те же поля, что и при создании — UC8, шаг 4).
4. Пользователь нажимает «Сохранить».
5. Система автоматически пересчитывает характеристики и расстояния (если изменились описание или координаты).
6. Система перенаправляет на обновлённую страницу деталей участка.

**Альтернативный сценарий:**
- *1a.* Пользователь не является владельцем участка → кнопка «Редактировать» не отображается.
- *4a.* Не изменено ни одно поле → сервер возвращает ошибку «Нет полей для обновления».
- *5a.* Ошибка сервера → система показывает сообщение об ошибке, данные не сохраняются.

**Результат:** Данные участка обновлены; характеристики и баллы пересчитаны при необходимости.

## UC10: Удаление своего участка

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Пользователь |

**Предусловие:** Пользователь авторизован; участок принадлежит этому пользователю; пользователь находится на странице деталей своего участка (UC5).

**Основной сценарий:**
1. Пользователь нажимает кнопку «Удалить».
2. Система отображает диалог подтверждения: «Удалить это объявление?».
3. Пользователь подтверждает.
4. Система удаляет участок из базы данных.
5. Система перенаправляет пользователя на страницу каталога.

**Альтернативный сценарий:**
- *1a.* Пользователь не является владельцем → кнопка «Удалить» не отображается.
- *3a.* Пользователь отменяет операцию → удаление не происходит, страница остаётся без изменений.
- *4a.* Участок уже удалён → сервер возвращает 404, система показывает ошибку.
- *4b.* Сервер недоступен → система показывает сообщение об ошибке.

**Результат:** Участок удалён из базы данных и больше не отображается в каталоге и на карте.

|------|----------|

## UC12: Импорт данных

| Поле | Значение |
|------|----------|
| **Действующее лицо** | Администратор |

**Предусловие:** Администратор авторизован; находится на странице «Панель данных»; есть корректный JSON-файл.

**Основной сценарий:**
1. Администратор нажимает «Импорт JSON» и выбирает `.json`-файл через диалог выбора файла.
2. Система распознаёт формат: массив объектов, либо объект с ключом `"plots"` или `"data"`.
3. Система загружает данные на сервер.
4. Сервер для каждого участка автоматически рассчитывает характеристики, эмбеддинги, расстояния и баллы.
5. Система показывает сообщение: «Импортировано N записей».
6. Таблица статистики обновляется.

**Альтернативный сценарий:**
- *2a.* Некорректный формат файла → система показывает ошибку парсинга.
- *4a.* Ошибка сервера при обработке → система показывает сообщение об ошибке.

**Результат:** Новые участки добавлены в базу с автоматически рассчитанными метриками.

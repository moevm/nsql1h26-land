# Аналитика земельных участков Avito

Модуль аналитики для сервиса покупки и продажи земельных участков в Санкт-Петербурге и Ленинградской области.  
Получает объявления с Авито, обогащает каждое объективными метриками (инфраструктура, экология, семантические фичи, цена) и ранжирует по комбинированному скору.

---

## Описание приложения

Приложение — **сервис для покупки и продажи земельных участков** с встроенной аналитикой. Данные берутся с Авито, обогащаются объективными метриками и показываются пользователю в удобном интерфейсе.


## Пользовательские сценарии

### Сценарий 1: Покупатель ищет участок по запросу

**Актор:** Покупатель  
**Цель:** Найти лучший участок для строительства дома

1. Открывает каталог участков.
2. Вводит запрос: *«тихий участок ИЖС с газом рядом с метро»*.
3. Настраивает фильтры: цена 1–5 млн ₽, площадь 8–15 соток.
4. Система запускает пайплайн:
   - Фильтрует по жёстким критериям (цена, площадь)
   - BM25 находит текстово релевантные объявления
   - Комбинированный скор (`α·quality + β·bm25`) отбирает лучших кандидатов
   - Jina Reranker семантически переранжирует финальный топ
5. Видит ранжированный список с тегами: `ИЖС`, `газ`, `электричество`, `тихое место`.
6. Кликает на карточку → детальная аналитика со скорами и расстояниями.
7. Нажимает «Связаться с продавцом» → переходит к сделке.

### Сценарий 2: Покупатель сравнивает участки без запроса

**Актор:** Покупатель  
**Цель:** Увидеть объективный рейтинг участков в районе

1. Открывает каталог **без текстового запроса**.
2. Выбирает район «Петергоф» и площадь от 6 соток.
3. Система показывает все участки, отсортированные по `total_score` (качество = инфраструктура + экология + фичи + цена). BM25 и Jina не применяются.
4. Сравнивает топ-5 по разбивке скоров в детальных карточках.
5. Добавляет понравившиеся в избранное.

### Сценарий 3: Покупка участка

**Актор:** Покупатель  
**Цель:** Купить выбранный участок

1. Находит подходящий участок через сценарий 1 или 2.
2. Изучает карточку: скоры, фичи, расстояния до инфраструктуры, фото.
3. Проверяет тег `документы готовы` — система подсказывает готовность к сделке.
4. Нажимает **«Связаться с продавцом»** → переходит на Авито или в чат.
5. Договаривается о просмотре и условиях.
6. Сервис сохраняет историю просмотренных и избранных участков.

### Сценарий 4: Продажа участка

**Актор:** Продавец  
**Цель:** Разместить объявление и продать по лучшей цене

1. Нажимает «Продать участок» → создаёт объявление (или импортирует с Авито).
2. Система автоматически анализирует описание:
   - Определяет категорию (ИЖС/СНТ), коммуникации, особенности.
   - Рассчитывает скор качества и позицию в рейтинге.
3. Продавец видит **аналитику конкурентов**: средняя цена за сотку в районе, количество похожих участков.
4. Получает **рекомендации**: какие фичи добавить в описание, оптимальная цена.
5. После правок перезапускает аналитику и видит улучшение позиции.

### Сценарий 5: Продавец оценивает конкурентоспособность

**Актор:** Продавец  
**Цель:** Понять, как улучшить объявление

1. Находит свой участок в рейтинге (по ID или адресу).
2. Видит позицию: *34 из 247* и разбивку по 4 факторам.
3. Замечает: характеристики 0.28 — мало фич обнаружено в описании.
4. Система подсказывает: *«Добавьте в описание: электричество, водоснабжение»*.
5. Дополняет текст объявления → score характеристик растёт с 0.28 до 0.55.

### Сценарий 6: Импорт данных

**Актор:** Администратор  

1. Загружает свежий JSON-файл с объявлениями с Авито.
2. Запускает `python pipeline.py --no-cache` для полного пересчёта.
3. Обогащение занимает ~2–3 минуты. Результат кэшируется.

### Сценарий 7: Экспорт аналитики

**Актор:** Покупатель / Аналитик  

1. Настраивает фильтры, запускает аналитику.
2. Экспортирует результаты в JSON/CSV с полной разбивкой скоров.
3. Использует для офлайн-анализа, отчётов, принятия решений.


| Параметр | Значение | Описание |
|----------|----------|----------|
| `α` (ALPHA) | 0.6 | Вес качества участка (инфры, экологии, фич, цены) |
| `β` (BETA) | 0.4 | Вес текстовой релевантности (BM25) |

- `total_score_norm` — min-max нормализованный `total_score` по выборке BM25
- `bm25_score_norm` — min-max нормализованный `bm25_score`
- Настраиваются в `config.py`. Сумма α + β = 1.0.

**Зачем:** combined score объединяет объективное качество участка с текстовой релевантностью запросу. Jina Reranker затем семантически переранжирует топ по combined — получая на вход уже «хорошие» участки, а не только текстово похожие.

**Без запроса:** `combined_score = total_score`, Jina не применяется.

### Веса качества (total_score)

| Фактор | Вес | Описание |
|--------|-----|----------|
| `infra` | 0.35 | Близость к метро, больнице, школе |
| `negative` | 0.25 | Удалённость от свалок, промзон, КАД |
| `features` | 0.25 | Семантические фичи из описания |
| `price` | 0.15 | Нормализованная цена за сотку (ниже = лучше) |

### Семантические фичи (15 штук)

Модель `paraphrase-multilingual-MiniLM-L12-v2` определяет характеристики по описанию через cosine similarity:

| Фича | Описание | Вес |
|------|----------|-----|
| `is_izhs` | ИЖС | 0.35 |
| `has_house` | Дом / постройки | 0.30 |
| `has_gas` | Газ подведён | 0.25 |
| `has_electricity` | Электричество | 0.20 |
| `has_water` | Водоснабжение | 0.20 |
| `has_communications` | Все коммуникации | 0.20 |
| `is_quiet` | Тихое место | 0.20 |
| `has_sewage` | Канализация / септик | 0.15 |
| `has_road` | Хороший подъезд | 0.15 |
| `documents_ready` | Документы готовы | 0.15 |
| `has_fence` | Огорожен | 0.10 |
| `flat_terrain` | Ровный участок | 0.10 |
| `has_forest` | Лес рядом | 0.10 |
| `near_river` | Водоём рядом | 0.10 |
| `is_snt` | СНТ / ДНП | 0.10 |

Порог обнаружения: cosine similarity ≥ 0.30.

## Требования

- Python 3.11+

- Jina API ключ (для Reranker, опционально)

## Локальный запуск

### 1. Установка

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux/macOS:
source .venv/bin/activate

pip install -r avito/requirements.txt
```

### 2. Настройка ключей

Создайте `.env` в корне проекта:

```env
JINA_API_KEY=your_jina_api_key_here
```

### 3. Данные

```
data/
└── avito_20260301_140233.json
```

Формат записи:

```json
{
  "id": 7547791865,
  "title": "Участок 4 сот. (СНТ, ДНП)",
  "description": "Продается участок 4 сотки...",
  "price": 1400000,
  "lat": 59.837,
  "lng": 29.884,
  "locationName": "Петергоф",
  "addressFull": "Ленинградская область, Ломоносовский район...",
  "geoReferences": "д. Низино, СНТ Сад-2",
  "url": "https://www.avito.ru/..."
}
```

### 4. Запуск

```bash
cd avito

# Топ-20 по quality (без запроса)
python pipeline.py

# С запросом → BM25 + Combined + Jina
python pipeline.py --query "тихий участок с домом и газом рядом с метро" --top 10

# Без Jina (финал по combined score)
python pipeline.py --query "ИЖС лес 10 соток" --top 5 --skip-jina

# С фильтрами
python pipeline.py --min-price 1000000 --max-price 5000000 --min-area 8 --max-area 20

# По району
python pipeline.py --location "Петергоф" --top 10

# Пересчитать всё
python pipeline.py --no-cache
```

### Все аргументы

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `--data` | `data/avito_*.json` | Путь к JSON-файлу |
| `--query`, `-q` | — | Текстовый запрос (BM25 + Combined + Jina) |
| `--top`, `-n` | `20` | Количество результатов |
| `--min-price` | — | Минимальная цена, ₽ |
| `--max-price` | — | Максимальная цена, ₽ |
| `--min-area` | — | Минимальная площадь, соток |
| `--max-area` | — | Максимальная площадь, соток |
| `--location` | — | Район / город |
| `--skip-jina` | `false` | Пропустить Jina (финал по combined) |
| `--no-cache` | `false` | Пересчитать с нуля |

## Настройка весов (config.py)

### Комбинированный скор

```python
ALPHA = 0.6   # вес качества участка (total_score)
BETA  = 0.4   # вес текстовой релевантности (BM25)
```

Варианты:

| ALPHA | BETA | Поведение |
|-------|------|-----------|
| 1.0 | 0.0 | Только качество, запрос → фильтрация, не ранжирование |
| 0.5 | 0.5 | Баланс |
| 0.0 | 1.0 | Только текстовая релевантность (BM25 решает всё) |

### Веса качества

```python
WEIGHTS = {
    "infra":    0.35,
    "negative": 0.25,
    "features": 0.25,
    "price":    0.15,
}
```

## Кэширование

Результаты обогащения (фичи + гео) после первого запуска сохраняются в `data/enriched_cache.json`.  
Повторные запуски загружают кэш и пропускают этапы 2–3 (~150 сек экономии).

Сброс: удалите файл или `--no-cache`.

## Структура проекта

```

├── pipeline.py          # Точка входа (8 этапов)
├── config.py            # Веса, координаты, α/β

├── feature_extractor.py # Семантические фичи (sentence-transformers)

├── scoring.py           # total_score (quality)
├── bm25_ranker.py       # BM25 pre-ranking
├── jina_reranker.py     # Jina Reranker API
├── requirements.txt     # Зависимости
└── Dockerfile           # Контейнеризация

data/
├── avito_*.json         # Входные данные
└── enriched_cache.json  # Кэш (генерируется)
```
